# 휴가관리 — 화면 설계

기존 Hub 디자인 시스템(`docs/conventions.md`)을 그대로 쓴다: 전체 inline style,
배경 `#f8f8f7`, 카드 `#fff` + `1px solid #e8e8e6` + radius 8, 헤더 바(56px),
`primaryBtn`(#111)/`outlineBtn` 패턴, 테이블 헤더 `#f4f4f2` sticky.

## 페이지 구조 (`/leave`)

```
┌ 헤더 바 ──────────────────────────────────────────────────────┐
│ 휴가관리        [공휴일 관리] [연차 설정] [+ 휴가 추가]           │
├ 툴바 ─────────────────────────────────────────────────────────┤
│ ‹ 2026년 ›   [직원명 검색____]  [직급 ▾]  [☑ 재직자만]          │
├ 메인 테이블 ───────────────────────────────────────────────────┤
│ 직급│직원명│입사일│부여 ┃1월…3월┊4월…6월┊7월…9월┊10월…12월┃사용│잔여│
│ (좌측 고정)          ┃   가로 스크롤 영역(분기 구분선)      ┃(우측 고정)│
├ 상세이력 (직원 행 선택 시) ────────────────────────────────────┤
│ [요약 카드: 직원명·직급·부여·사용·잔여]                          │
│ [필터: 유형 ▾ / 차감여부 ▾ / 정렬 ▾]                            │
│ [이력 테이블: 유형|시작일|종료일|기간|차감|메모|등록일|수정|삭제]   │
└───────────────────────────────────────────────────────────────┘
```

## 메인 테이블

열: 직급 · 직원명 · 입사일 · 연간 부여 연차 · 1월~12월 · 연간 사용일수 · 잔여 연차일수.

- **고정 열**: 직급·직원명·입사일·부여(좌측 `position: sticky; left`),
  사용·잔여(우측 `position: sticky; right`). 헤더 행은 `top: 0` sticky.
  MonthGrid/ProjectGrid의 sticky 패턴 재사용 (배경색을 줘야 스크롤 시 겹침이 안 보인다).
- **분기 구분선**: 3월|4월, 6월|7월, 9월|12월 경계에 `borderLeft: '2px solid #ccc'`
  (연장근무 그리드의 "달력 월 경계" 처리와 같은 방식). 헤더에 1/4~4/4분기 상단 그룹
  행은 두지 않는다 — 구분선만으로 충분하고 sticky 2단 헤더는 복잡도만 는다.
- **월 셀**: 그 달 실제 차감일수 합 (`1`, `0.5`, `3.5`). 0이면 빈 칸.
  값이 있는 셀은 엑셀 원본처럼 앰버 배경(`#fffbeb` + `#b45309` 텍스트)으로 구분.
  클릭 → 월 셀 상세 팝오버 (읽기 전용 — 수정은 이력 테이블의 수정 버튼으로만).
- **잔여 경고**: 잔여 ≤ 0이면 잔여 셀을 위험색(`#b91c1c`/`#fef2f2`)으로.
- **행 클릭** → 아래 상세이력 영역에 그 직원 표시 (재클릭 시 해제).
- 직원 정렬은 `overtime_employees.sort_order`, 검색·직급 필터·재직자만 보기는
  클라이언트 필터 (projects 페이지 패턴).
- 퇴사자는 "재직자만" 해제 시 표시하되 이름 옆 `(퇴사)` 표기 — 과거 기록 보존.

## 연차 설정 모달 (`BalanceManagerModal`)

ProjectManagerModal 패턴(행 단위 onBlur 즉시저장)을 따른다. 상단에 대상 연도 표시.

행: 직원명 · **입사일** · **퇴사일** · 기본 부여(숫자, 0.5 단위) · 조정일수(±) ·
조정 사유 · 최종(=부여+조정, 자동 계산 표시) · 수정일시.

- 입사일/퇴사일은 `overtime_employees` 공용 컬럼을 여기서 편집한다 — 연장근무의
  직원 관리 화면은 수정 금지 제약이 있어 휴가관리 쪽에 편집 UI를 둔다.
- 부여/조정 변경 시: 이미 사용한 일수 > 변경 후 최종이면 경고 문구 표시(저장은 허용).
- 저장할 때마다 `annual_leave_balance_history`에 변경 전/후 기록.
- 해당 연도 balance 행이 없는 직원은 빈 값으로 표시하고 첫 입력 시 insert.

## 휴가 추가/수정 모달 (`LeaveRecordModal`)

프로젝트 추가 모달과 같은 중앙 모달(#111 헤더). 필드:

- 직원(드롭다운, 재직자 + 수정 시 해당 직원) · 휴가 유형(드롭다운)
- 시작일 · 종료일 (date input)
- 시작일 단위 / 종료일 단위: 전일·오전 반차·오후 반차 (같은 날이면 단위 하나만 표시)
- **자동 계산 표시줄**: `전체 3박 4일 · 실제 차감 2일 (주말 2일 제외)` — 입력 변경 시 실시간
- 메모 · [저장] [취소], 수정 모드면 하단에 검증 경고 영역

유형에서 "오전 반차"/"오후 반차"를 고르면 종료일=시작일로 잠그고 단위를 자동 세팅한다
(유형은 분류·차감여부용, 실제 계산은 날짜별 단위 기준 — 04 문서).

**공동 등록** (사용자 요청으로 추가 — 회사 공동 연차용): 신규 등록 시 모달 상단의
[개별 등록 | 공동 등록] 토글. 공동 모드에서는 직원 드롭다운 대신 재직자 체크 목록
(기본 전원 선택, 전체 선택/해제 버튼)이 나오고, 저장하면 선택된 직원 수만큼 개별
레코드를 만든다 — 저장 단위는 그대로라 이후 수정/삭제는 직원별로 한다.
입사 전·퇴사 후·기존 휴가와 겹치는 직원은 자동으로 건너뛰고 결과를 알려주며,
잔여 초과 직원은 한 번에 모아 경고 후 confirm으로 진행한다. 수정 모드에는 없다.

## 월 셀 상세 팝오버 (`MonthCellPopover`)

연장근무 OvertimeEntryPopover처럼 셀 근처에 뜨는 작은 창. 읽기 전용.

```
김○○ · 2026년 8월 · 사용 3일
─────────────────────────
8/3        연차        1일
8/10       오후 반차    0.5일
8/17~8/18  연차        1.5일
```

## 공휴일 관리 모달 (`HolidayManagerModal`)

연도별 목록(날짜·이름·구분: 법정공휴일/회사휴무) + 추가/수정/삭제.
상단에 「{연도} 공휴일 인터넷에서 불러오기」 버튼 — 기존 `/api/holidays` 프록시로
가져와 upsert (오프라인이면 실패 안내만, 기존 데이터는 유지). 상세는 04 문서.

## 휴가 유형 관리

MVP는 기본 8종 시드로 시작하되, 공휴일 관리 모달과 같은 수준의 단순 관리 UI를
연차 설정 모달 하단 탭 또는 별도 모달로 제공: 이름·차감 여부·기본 차감 단위·
활성화·표시 순서 편집. 이미 사용된 유형은 삭제 대신 비활성화(연장근무 소프트 삭제 원칙).
