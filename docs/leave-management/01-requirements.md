# 휴가관리 — 요구사항 정리

> 근태관리 하위의 독립 메뉴. 팀원별 연간 부여 연차, 월별 사용 연차, 누적 사용일수,
> 잔여 연차, 휴가 사용 이력을 한 화면에서 관리한다.
> 원본 요구사항(사용자 마스터 프롬프트)을 이 프로젝트 구조에 맞게 정리한 문서.

## 기본 원칙

- **연장근무와 독립된 페이지** — 연장근무 페이지 내부 탭/카드/하단 테이블로 넣지 않는다.
- **직원 정보는 `overtime_employees`를 공용으로 사용** — 중복 직원 테이블을 만들지 않는다.
- 휴가 기록·연차 정보는 별도 DB 테이블 (`docs/leave-management/03-data-model.md`).
- **기존 연장근무 기능·데이터·화면에 영향 없음** — 연장근무 쪽 파일은 수정하지 않는다
  (직원 테이블에 컬럼 추가만 예외 — nullable이라 기존 기능에 영향 없음).
- 기존 Hub의 레이아웃·폰트·색상·버튼·테이블 디자인 유지 (`docs/conventions.md`).
- 단일 사용자용이라는 기존 전제 유지 — 승인/결재/동시편집 없음.
- 기술 스택·마이그레이션 방식 그대로: Next.js App Router + Supabase,
  `supabase/migration_*.sql` 파일 + MCP `apply_migration`.

## 메뉴/라우트

- 사이드바 「사무 보조 → 근태관리」 하위: `연장근무` / **`휴가관리`** (연장근무 바로 아래,
  기술인 주소록 바로 위).
- 라우트: **`/leave`** — 이 프로젝트는 `/attendance/*` 같은 그룹 경로를 쓰지 않고
  `app/(dashboard)/` 아래 평면 라우트(`/overtime`, `/projects`, ...)를 쓴다. 기존 `/overtime`을
  옮기면 링크가 깨지므로 휴가관리도 같은 방식으로 `/leave`에 둔다.
- 실제 클릭 가능 메뉴(준비중 아님), 활성화 스타일은 기존 자식 메뉴 방식 그대로
  (`pathname.startsWith(href)` — `/overtime`과 `/leave`는 접두사가 달라 독립 활성화됨).
- 사람별 메뉴 숨김: `lib/menuConfig.ts`의 `RESTRICTABLE_MENU_ITEMS`에 `key: 'leave'` 추가.

## 기능 요약 (상세는 02/04 문서)

| 영역 | 내용 |
|---|---|
| 메인 테이블 | 직원 × 월(1~12) 그리드. 부여연차·월별 사용·연간 사용·잔여 표시. 분기 구분선 |
| 연차 설정 | 연도별·직원별 기본 부여일수 + 조정일수(+사유). 수정 이력 저장. 소수점 허용 |
| 휴가 추가 | 유형·기간·시작/종료일 단위(전일/오전/오후) 입력 → 몇 박 며칠 + 실제 차감일수 자동 계산 |
| 일수 계산 | 주말·법정공휴일·회사휴무 차감 제외, 반차 0.5일, 유형별 차감 여부 |
| 집계 | 월별/연간 사용일수·잔여 연차는 원본 기록에서 항상 재계산 (저장 안 함) |
| 상세이력 | 직원 행 클릭 → 요약 카드 + 휴가 이력 테이블 (수정/삭제) |
| 월 셀 상세 | 월 칸 클릭 → 그 달의 개별 휴가 내역 (읽기 전용) |
| 검증 | 기간 역전·입퇴사일 범위·중복·잔여 초과(경고 후 허용) 등 |

## 1차 MVP 범위

포함: 사이드바 메뉴, `/leave` 페이지, 직원 연동(+입사일/퇴사일 컬럼 추가), 연도 선택,
연차 부여/조정 입력·수정(+이력), 휴가 유형 기본 8종 시드 + 관리, 휴가 등록/수정/삭제,
몇 박 며칠 계산, 주말·공휴일 제외, 반차 0.5일, 월별/연간/잔여 자동 집계, 직원별 이력,
월 셀 상세보기, 중복 검증, 잔여 초과 경고, 공휴일/회사휴무 관리(+시드).

제외(추후 확장): 직원별 로그인, 신청/승인/결재, 알림, 모바일 전용 화면, 외부 캘린더 연동,
연차 사용 촉진, 신청서 출력, 동시 사용, 근속연수별 자동 발생, Excel/PDF 내보내기,
soft delete(변경/삭제 이력) — MVP는 일반 삭제(15절 사용자 승인).
