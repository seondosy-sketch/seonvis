# 기술인 출퇴근부 — 테스트 계획

> 마스터 프롬프트 15번의 필수 테스트 목록을 그대로 카테고리별로 정리한다. 이 코드베이스에는 현재
> 자동 테스트 러너(Jest 등)가 설정돼 있지 않다(`package.json`에 test 스크립트 없음, Confirmed) —
> Phase 1에서 최소 테스트 러너 도입 여부를 함께 결정해야 한다(Open Question, 아래 §0).

---

## 0. Open Question — 테스트 인프라

이 저장소에는 테스트 프레임워크가 전혀 설치돼 있지 않다(`vitest`/`jest` 없음). 아래 테스트 목록을 실행하려면 최소한의 유닛 테스트 러너 도입이 선행돼야 한다. Proposed: 순수 함수 위주(기간 계산, 검증 로직, 비고 조립)라 `vitest` 최소 설정으로 충분 — 어떤 러너를 쓸지, 혹은 수동 확인으로 대체할지 사용자 확인 필요.

## 3. 마감 (Phase 4)

| # | 테스트 |
|---|---|
| 3.1 | 마감 후 해당 월 출근기록 수정이 차단됨 |
| 3.2 | 마감취소 권한(`can_close_attendance` 또는 확정된 방식)이 없는 사용자는 취소 버튼이 비활성/숨김 |
| 3.3 | 마감취소 시 사유 입력 없이는 제출되지 않음(필수 검증) + `attendance_audit_log`에 기록됨 |
| 3.4 | 마감 시 `attendance_closure_snapshot_rows`가 그 시점 데이터와 정확히 일치하도록 생성됨 |
| 3.5 | 마감 이후 Project List(`projects`, `project_participants`)를 수정해도 이미 생성된 스냅샷 조회 결과가 달라지지 않음(핵심 요구사항 — 반드시 확인) |
| 3.6 | 마감 전 검증에서 오류(중복 기록, 필수 식별정보 누락)와 경고(면접일 미입력, 비고 미작성)가 올바르게 구분됨 |

## 4. 출력 (Phase 5, 6)

| # | 테스트 |
|---|---|
| 4.1 | 생성된 xlsx의 날짜 열 개수가 실제 기간 일수와 일치 |
| 4.2 | 출근일에 지정된 배경색(fill)이 적용됨 + 셀 값 `1`이 함께 기입됨 |
| 4.3 | 연간 출력에서 월 경계(회사 기준월)가 정확한 위치에 그려짐 |
| 4.4 | 프로젝트명·직책·분야·성명이 정확히 병합·표시됨 |
| 4.5 | 비고가 `wrapText`로 줄바꿈되어 출력되고 행 높이가 내용에 맞게 조정됨 |
| 4.6 | 동일 프로젝트의 다수 기술인이 병합 셀 아래 정상적으로 나열됨(페이지 경계에서 블록이 분리되지 않는지 포함) |
| 4.7 | 연간 출력이 12개 회사 기준월로 정확히 구분됨(전월21~당월20 경계) |
| 4.8 | 미마감 월이 섞인 연간 출력에서 경고 표시(확정본/임시본 구분)가 규칙대로 적용됨 |

## 5. 회귀 테스트 (모든 Phase 공통, Phase 7에서 최종 실행)

| # | 대상 | 확인 항목 |
|---|---|---|
| 5.1 | Project List | 신규 컬럼(`project_participants` 등) 추가가 기존 CRUD·상태계산(`computeProjectStatus`)·CSV 내보내기에 영향 없음 |
| 5.2 | 연장근무 | `lib/overtime/summary.ts`에 함수 추가/이동 시 기존 `payPeriodDays`/`payPeriodRange`/`currentPayPeriod` import 경로와 동작이 그대로 유지됨(그리드·대시보드·출력 페이지 전부 재확인) |

| 5.5 | 권한 체계 | `allowed_users`에 컬럼 추가 후 기존 `menu_permissions` 기반 사이드바 숨김/읽기전용 로직이 정상 동작 |

## 6. 수동 확인 필요 항목 (자동화 어려움)

- 실제 Excel 출력물을 Excel/LibreOffice에서 직접 열어 음영·병합·인쇄영역·배율·반복행이 화면과 동일하게 보이는지 육안 확인(마스터 프롬프트 16번 완료 기준).
- 모바일 화면에서의 축소 안내 문구 노출.
- 대용량(연간 376열 × 실제 프로젝트 수) 상태에서 그리드 스크롤 체감 성능.
