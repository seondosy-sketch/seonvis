# 기술인 출근부 — 출력(Export) 설계

> 전제: `01-current-analysis.md` §4에서 확인했듯 이 코드베이스에는 실제 xlsx 바이너리 생성 라이브러리가 없다.
> 이번 기능은 마스터 프롬프트가 "XLSX 최소 지원"을 명시했으므로, 이 기능에서 처음으로 신규 의존성을 들여온다.

---

## 1. 라이브러리 선택 (Proposed)

| 후보 | 비고 |
|---|---|
| **exceljs (채택 제안)** | 셀 병합, 배경색, 테두리, 열 너비/행 높이, 인쇄영역, 페이지 설정(가로/세로, 배율, 반복 행/열), 틀고정까지 전부 API로 지원. Node 서버(API Route)에서 버퍼로 생성해 그대로 응답 가능 — 이 코드베이스의 API Route 패턴(`app/api/hwpx/route.ts`)과 동일한 형태로 `app/api/attendance/export/route.ts`를 만들 수 있다 |
| xlsx(SheetJS) | 커뮤니티판은 셀 서식(배경색 등) 쓰기 기능이 제한적 — 이번 요구사항(음영 재현)에 부적합 |
| 브라우저에서 클라이언트 생성 | 서버 없이 가능하지만, 마감 스냅샷(§`03-data-model.md`)을 서버에서 조립하는 흐름과 맞지 않고, 대용량(연간 376열) 생성은 서버에서 하는 편이 안전 |

**Proposed**: `exceljs`를 신규 의존성으로 추가하고, 서버 API Route(`app/api/attendance/export/route.ts`)에서 워크북을 생성해 다운로드 응답. `docs/overtime.md` 8단계가 "API route는 향후 진짜 바이너리 생성이 필요해질 때 만든다"고 명시적으로 미뤄둔 지점이 바로 지금이다.

**Open Question**: exceljs 같은 신규 npm 의존성 추가에 동의하는지(패키지 크기·유지보수 부담 고려) — 동의하지 않으면 CSV+브라우저 인쇄로 1차 범위를 축소하는 대안도 가능(연장근무가 8단계에서 택한 방식과 동일, 다만 마스터 프롬프트가 "최소 Excel 지원"을 명시했으므로 권장하지 않음).

## 2. 월별 출력 레이아웃 (마스터 프롬프트 8번 — Confirmed 항목 매핑)

### 2.1 시트 구성
```
행1: 제목 "기술인 출근명부(2026년 8월분)"          ← 병합, 16pt bold, 첨부 엑셀과 동일 폰트(맑은 고딕)
행2: 대상기간 "2026.07.21 ~ 2026.08.20"    출력일: 2026-08-21   마감: 완료(2026-08-20 홍길동)
행3: (빈 줄)
행4: 프로젝트 | 직책 | 분야 | 성명 | [날짜 헤더 31개] | 출근일수 | 비고
행5~: 데이터
```
- 날짜 헤더 2단(월 병합 + 일자, 첨부 엑셀 §7.2 구조 재현) + **요일 1단 추가**(엑셀에는 없었으나 마스터 프롬프트가 요구 — `01-current-analysis.md` §7.4/`02-requirements.md` §4 확인).
- 토·일 헤더는 배경색 구분(연장근무 결정사항 문서의 기존 팔레트 재사용, 새 색 도입 안 함), 공휴일은 `holidays`/`/api/holidays` 조회 결과로 별도 배경.

### 2.2 프로젝트명 표현 — 엑셀 출력에서는 병합 채택 (Confirmed 방침)
화면(`04-ui-flow.md` §2.2)과 동일하게 **엑셀 출력물에서는 프로젝트명·직책 열을 그 프로젝트의 기술인 행 수만큼 세로 병합**한다. 원본 엑셀은 병합이 아니라 빈 칸 방식이었지만(`01-current-analysis.md` §7.3), 마스터 프롬프트가 "같은 프로젝트명 여러 행은 병합 또는 반복 표시 중 기존과 유사한 방식"이라고 선택지를 열어뒀고, 병합 쪽이 인쇄 시 가독성이 더 높고 exceljs로 구현이 간단하다.

### 2.3 출근 셀 (마스터 프롬프트 "값 vs 음영" 요구 — Confirmed 방침)
- 배경색: 출근일에 회색 계열 solid fill(첨부 엑셀 theme 색조 근사치로 통일 — 정확한 브랜드 색이 아니었으므로 웹 접근성 고려해 `#D9D9D9` 계열로 고정 채택).
- 셀 값: **접근성을 위해 `1`을 명시적으로 기입**(Proposed — `○`/`출근` 대신 `1`을 택한 이유: 우측 "출근일수" 열이 `SUM()` 수식으로 그 값을 그대로 합산할 수 있어, 화면 표시와 계산이 같은 데이터에서 나온다는 원칙 유지. `anthropic-skills:xlsx`의 "하드코딩 대신 수식" 가이드와도 일치).
- 미출근: 빈 값 + 기본 배경.
- 프로젝트 변경이력으로 인한 특이일(공고취소·재공고 등, `03-data-model.md` §5)이 그 날짜와 정확히 겹치면 셀에 작은 각주 기호(`*`) 병기 + 시트 하단에 각주 목록(첨부 엑셀에서 실측된 "날짜 칸에 직접 텍스트를 적는" 관행을 완전히 대체하는 정상화된 표현, `01-current-analysis.md` §7.4).

### 2.4 인쇄 설정 (마스터 프롬프트 8번 — Confirmed 요구, 첨부 엑셀에는 부재했던 것을 신규 설계 — `01-current-analysis.md` §7.6)
| 항목 | 값 |
|---|---|
| 방향 | 가로(landscape) |
| 용지 | A4 |
| 배율 | `fitToWidth: 1`, `fitToHeight: 0`(세로는 자연스럽게 여러 페이지로 넘어가되 가로는 항상 1페이지 폭에 맞춤 — exceljs `pageSetup.fitToPage`) |
| 반복 열 | A~D열(프로젝트/직책/분야/성명) — exceljs `worksheet.pageSetup.printTitlesColumn` |
| 반복 행 | 1~4행(제목~날짜 헤더) — `printTitlesRow` |
| 여백 | 상하좌우 좁게(0.5in), 머리글/바닥글 0.3in |
| 페이지 번호 | 바닥글 우측에 "N/M 페이지" |
| 행 분리 방지 | 한 프로젝트의 참여기술인 행이 페이지 경계에서 끊기지 않도록 exceljs의 수동 페이지 나누기(`worksheet.rowBreaks`)를 프로젝트 블록 경계에만 삽입 — 그리드가 만들어질 때 블록 크기를 계산해 페이지당 남은 행 수보다 큰 블록이면 그 블록 시작 전에 강제 개행 |
| 월 구분(연간 출력 한정) | 회계 월이 바뀌는 열 경계에 세로 굵은 테두리 |

### 2.5 비고 (마스터 프롬프트 — Confirmed)
- 셀 `alignment.wrapText = true`, 열 너비는 첨부 엑셀 실측값(11.875, `01-current-analysis.md` §7.2)을 그대로 채택.
- 내용이 많은 행은 `worksheet.getRow(r).height`를 텍스트 줄 수에 비례해 자동 계산(고정 31.5 대신 동적 — 첨부 엑셀은 전 행 고정 31.5였지만 비고가 길어지는 경우까지 감안해 정상화).

## 3. 연간 통합 출력 (마스터 프롬프트 9번)

- 열 구성은 월별과 동일 + 날짜 범위만 전년도 12/21~해당연도 12/20(§`02-requirements.md` #1 Open Question 결과에 따라 12/31까지 확장될 수 있음).
- 데이터 소스: `attendance_closure_snapshot_rows` 12개월치 UNION(§`03-data-model.md` §4) — Project List가 그 사이 바뀌어도 흔들리지 않음.
- 미마감 월이 섞여 있으면(확정본이 아니라 임시본 요청 시) 그 달의 날짜 열 헤더 배경에 경고색 줄무늬 + 시트 상단에 "○월 임시 데이터 포함" 안내 셀 삽입.
- 파일명(Proposed): `기술인출근부_연간_2026(확정).xlsx` / `..._임시.xlsx`로 확정/임시 구분.

## 4. PDF / 인쇄 미리보기 (마스터 프롬프트 8번 — "검토해라" 항목)

**Proposed**: 1차 구현에서는 exceljs로 만든 xlsx만 제공하고, PDF는 **브라우저 인쇄**(연장근무가 이미 쓰는 `window.print()` + `.no-print` 패턴, `01-current-analysis.md` §4.2)로 대체 — 별도 PDF 생성 라이브러리(예: puppeteer)는 서버 자원 부담이 커서 1차 범위 제외를 제안. 화면은 `/attendance/print`에 그리드와 동일한 데이터를 인쇄 전용 레이아웃(A4 가로, `@media print`)으로 렌더링.

**Open Question**: PDF를 서버에서 진짜 파일로 생성해야 하는 필수 요구사항인지, 위 "인쇄 미리보기+브라우저 PDF 저장"으로 충분한지 확인 필요.

## 5. 셀 값 저장 형식 요약 (마스터 프롬프트 8번 선택지 결정)

| 선택지 | 채택 여부 |
|---|---|
| `1` | ✅ 채택 — 수식 합산 가능, 접근성 확보 |
| `○` | 기각 — 합산 불가, `1`과 표현력 차이 없음 |
| `출근` | 기각 — 셀 폭 3.125(첨부 엑셀 실측)로는 텍스트가 넘침 |
| 빈 값 + 배경색만 | 기각(값 없이는 스크린리더/필터링 불가 — 마스터 프롬프트가 우려한 지점 그대로) |

화면 표시는 여전히 "빈 값 + 배경색"(엑셀 관행, `04-ui-flow.md`)이지만, **DB(`attendance_records`)와 엑셀 출력 셀 값은 항상 명시적**이라는 원칙은 동일하게 유지된다.
