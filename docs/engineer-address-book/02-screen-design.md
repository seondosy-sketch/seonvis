# 기술인 주소록 — 화면 설계

기존 Hub 디자인 시스템 그대로: inline style, 배경 `#f8f8f7`, 카드 `#fff`+`#e8e8e6` 테두리,
56px 헤더 바, `primaryBtn`/`outlineBtn`, 테이블 헤더 `#f4f4f2` sticky.

## 페이지 구조 (`/engineers`)

```
┌ 헤더 바 ──────────────────────────────────────────────┐
│ 기술인 주소록                    [전문분야 관리] [+ 기술인 추가] │
├ 요약 카드 ────────────────────────────────────────────┤
│ [전체 667명] [재직 N명] [비활성 N명] [즐겨찾기 N명]        │
├ 툴바 ─────────────────────────────────────────────────┤
│ [통합 검색__________] [재직▾][직위▾][전문분야▾][지역▾]     │
│ [☆ 즐겨찾기만] [정렬▾] [필터 초기화]                      │
├ 목록 테이블 ───────────────────────────────────────────┤
│ ☆│성명│직위│전문분야│핸드폰 ⧉│지역│주소(말줄임) ⧉│상태│관리 │
│ (성명 좌측 고정 · 헤더 상단 고정 · 행 클릭 → 상세 모달)     │
└───────────────────────────────────────────────────────┘
```

부제(헤더 아래 회색 12px): "기술인의 연락처와 기본정보를 검색하고 관리합니다."

## 목록 테이블

열: ☆(즐겨찾기 토글) · 성명 · 직위 · 전문분야(칩) · 핸드폰(+복사) · 지역 · 주소(말줄임+복사) ·
상태 · 관리(수정). 소속·이메일은 목록에서 빼고 상세/검색에서만 — 667행 × 12열은 과밀.

- 성명 열 `position: sticky; left: 0`(배경 필수), 헤더 행 `top: 0` sticky — 휴가관리
  LeaveYearTable과 같은 패턴.
- 주소는 `max-width` + ellipsis, `title` 속성으로 hover 전체 표시.
- 복사 버튼: 전화·주소 옆 작은 ⧉ 아이콘 버튼. `navigator.clipboard.writeText(표시값 그대로)`
  → 버튼 근처 "복사했습니다" 토스트 1.5초 (전화 클릭 복사는 오작동 위험으로 안 함 — 요구 9).
- 행 클릭 → 상세 모달. 복사/즐겨찾기 버튼은 `stopPropagation`.
- 비활성(퇴직 포함) 행은 `opacity: 0.5` + 상태 뱃지(재직 그린 / 퇴직 회색 / 비활성 회색).
- 667행 전체 렌더 — 이 규모는 React 테이블로 충분해서 페이지네이션/가상 스크롤 없음.
  (렌더가 늦어지면 그때 "상위 200건 + 더보기"로 바꾼다 — 04 문서)
- 검색 결과 없음: "검색 조건에 맞는 기술인이 없습니다."

## 상세 모달 (행 클릭)

projects 페이지의 툴팁 보기 모달 패턴(#111 헤더). 우측 고정 패널 대신 모달로 — 이 앱의
기존 상세 보기가 전부 모달이고, 1400px 컨테이너에서 상시 패널은 테이블 폭을 잡아먹는다.

```
┌ #111 헤더: 김영준  부장 · 건축/품질   ☆ [수정] [✕] ┐
│ 핸드폰   010-1234-5678  [복사]                    │
│ 이메일   xxx@xxx.com    [복사]   사무실 02-...     │
│ 주소     (전체 주소 표시)  [복사]                   │
│ 소속/직책/입사일/퇴사일/재직상태/비고/등록·수정일     │
│ 하단: [비활성화]  [완전 삭제(빨강)]                  │
└──────────────────────────────────────────────────┘
```

## 추가/수정 모달

섹션 2개(기본 정보 / 추가 정보 — 접기 없이 나열). 필수: 성명·직위·핸드폰·주소.

- 성명* · 직위*(드롭다운: 기존 직위 목록 + 직접 입력) · 소속 · 직책
- 전문분야: 칩 다중 선택(건축/토목/기계/전기/통신/소방/안전/품질/공정/사업관리/조경/기타)
- 핸드폰*(숫자만 쳐도 자동 하이픈) · 사무실 전화 · 이메일
- 지역(시·도 드롭다운) · 주소*(단일 텍스트 — 안 1, 06 문서 ④) · 비고
- 재직 상태(재직/퇴직/비활성) · 입사일 · 퇴사일 · ☆ 즐겨찾기
- 저장 시 검증·중복 경고는 04 문서.

## 전문분야 관리 모달

leave의 유형 관리와 같은 단순 모달: 이름·표시순서·활성화, 사용 중이면 삭제 대신 비활성화.

## 읽기 권한(menu_permissions: read)일 때

+ 추가/전문분야 관리 버튼, 목록의 수정·즐겨찾기 토글, 상세 모달의 수정/비활성화/삭제 숨김.
검색·필터·복사·상세 보기는 그대로.
