# 기술인 주소록 — 요구사항 정리

> 사이드바 「사무 보조 → 기술인 주소록」(현재 준비중)을 실제 기능으로 활성화.
> 엑셀 화면 복제가 아니라, 기술인을 빠르게 검색하고 연락처·주소를 즉시 확인/복사하는
> **검색형 주소록**으로 만든다.

## 원본 데이터 실태 (address book.xls, 2026-07 조사)

- 시트 1개("26.06.30 기준"), **667명** × 4열(성명·직위·핸드폰·주소).
- 직위 16종: 상무 410 / 이사 43 / 사원 42 / 전무 34 / 대리 27 / 부장 27 / 차장 22 /
  과장 18 / 부사장 16 / 부팀장 8 / 소장 6 / 사장 4 / 팀장 3 / 상무이사 3 / 총괄사장 1 /
  고문 1 (+이상치 "8" 1건 — 가져올 때 보정).
- 품질: 전화 결측 1, 주소 결측 0, **전화 중복 3, 동명이인 4** → 중복은 경고만 하고 허용해야
  하는 실데이터 근거. 전화는 이미 `010-XXXX-XXXX` 하이픈 포함 형식.
- **팀 내부 직원(11명) 명단이 아니라 외부 기술인력 풀**이다 → 기존 `overtime_employees`
  재사용 불가, 별도 테이블로 간다 (03 문서).

## 기본 원칙

- 독립 라우트 `/engineers` (이 프로젝트는 `app/(dashboard)/` 평면 라우트 — `/engineers/address-book`
  같은 중첩 대신 기존 규칙을 따름). 라우트 전용 컴포넌트는 `_components/` 콜로케이션.
- 기존 화면(프로젝트 List·연장근무·휴가관리)과 코드 독립, Hub 디자인 시스템 유지
  (`docs/conventions.md`).
- 기존 기능 영향: 사이드바(`app/sidebar.tsx`)와 `lib/menuConfig.ts` 항목 추가 2곳뿐.
- 권한: 방금 도입한 **항목별 권한(쓰기/읽기/숨김)** 을 그대로 탄다 — `menuConfig`에
  `engineers` 키 추가만 하면 관리자 화면에 자동 노출. 읽기 권한이면 추가/수정/비활성/삭제
  UI 숨김(복사·검색은 허용). 개인정보 표시는 내부 관리자용 전제로 전체 표시(22절 우선안).
- 삭제는 **비활성화 우선**(soft) — 완전 삭제는 상세에서만, 확인창 후.

## 기능 요약

| 영역 | 내용 |
|---|---|
| 통합 검색 | 성명·직위·전문분야·소속·전화·이메일·주소·비고 대상, 입력 즉시(debounce 250ms) |
| 필터 | 재직 상태 / 직위 / 전문분야 / 지역 / 즐겨찾기 — 동시 적용 + 초기화 버튼 |
| 목록 | 테이블(성명 좌측 고정, 헤더 상단 고정), 667행 전체 렌더(페이지네이션 없음 — 규모상 불필요) |
| 복사 | 전화·주소·이메일 복사 버튼(클립보드 + "복사했습니다" 토스트) |
| 상세 | 행 클릭 → 상세 모달(전체 주소·연락처·복사·수정·비활성화·삭제) |
| 추가/수정 | 모달 입력, 전화 자동 하이픈, 중복(전화/이메일) 경고 후 허용, 입력값 검증 |
| 초기 데이터 | 엑셀 667건을 **1회 시드로 직접 이관** (범용 가져오기 UI는 확장) |

## 1차 MVP 범위

포함: 메뉴 활성화, `/engineers` 페이지, 목록 테이블, 통합 검색+debounce, 재직/직위/전문분야/
지역/즐겨찾기 필터, 추가/수정/비활성화/삭제, 전화 자동 포맷+복사, 주소 복사, 상세 모달,
정렬(성명/직위순서/최근수정/즐겨찾기·재직 우선), 중복 경고, 입력값 검증, 전문분야 다중 지정
(+관리), 엑셀 667건 초기 시드, 항목별 권한 연동.

제외(추후 확장): 카드 보기, 범용 Excel 가져오기/내보내기 UI, 주소 검색 API(우편번호),
지도 검색 링크, 직위 순서 관리자 설정 화면(MVP는 코드 상수), 개인정보 마스킹,
프로젝트 참여이력·기술인 경력 DB 연동, 문자/이메일 연동, 내부 직원(employee_id) 연결.
