# 주요 기능

## 1. 메인 대시보드 (`/`)

**구성 (데스크톱 2×2 그리드 / 모바일 세로 스크롤)**
- **일정 캘린더**: 이번 주 기준 월 달력. 수행 프로젝트의 제출일·발표일·개찰일 표시. 메모 있으면 이벤트 칩에 `●` 표시 + hover 툴팁.
- **금주 일정**: 이번 주에 해당하는 일정만 요약 (제출/발표/개찰 구분).
- **미래봇**: Claude API 기반 챗봇. 프로젝트 조회·등록·수정 자연어 처리. DB 수정 전 confirm 단계 있음.
- **참고 자료**: 미래봇 대화 중 언급된 관련 프로젝트 카드 표시.

**데이터 소스 우선순위**
1. `performing_projects` 테이블 (해당 주 저장 데이터)
2. 없으면 `projects` 테이블 직접 (드랍/취소/수주 제외)

---

## 2. 프로젝트 List (`/projects`)

입찰 현황 전체 목록 테이블.

**핵심 기능**
- **상태 필터**: 전체/진행중/수주/탈락 탭
- **타입 필터**: 면접/SOQ/종심제/TP/PQ/기타
- **검색**: 용역명·발주처·단장·번호 실시간 검색
- **추가/수정**: 모달 폼 — `projects` + `project_tooltips` 동시 저장
- **툴팁 정보**: 용역명 클릭 → 공고 상세 모달
- **셀 메모**: 발주처·제출일·발표일·개찰일·참여사 컬럼에 메모 기능
  - 주황색 점(●) = 메모 있음 / 회색 점 = 없음
  - 클릭 → 팝업 에디터 (저장/삭제)
  - `project_notes` 테이블에 `upsert`
- **CSV 내보내기**

**상태 자동 계산 (`computeStatus`)**
```
status_override → 그 값
participants에 "드랍"/"드롭" → "취소"
evaluation === "선" → "수주"
result_score 또는 evaluation 비어있으면 → "진행중"
나머지 → "탈락"
```

## 3. 주간/월간 업무보고 (`/weekly`)

매주 업무 보고서를 작성하고 HWPX 파일로 내보내는 페이지.

**섹션 구성**
1. **수행 프로젝트**: 개찰 / 진행중 구분 테이블
2. **발주예상 프로젝트**: 이번 주 주목 프로젝트
3. **교육참가자**: 직책별 이름 자동 집계 (수동 수정 가능)
4. **기타**: 자유 입력

**자동 채우기 로직**
- `performing_projects`에 저장 데이터 있으면 → 로드 + **재분류** 적용
- 없으면 → `projects`에서 자동 생성

**프로젝트 분류 규칙 (`categorizeProject`, weekStart 기준)**
```
1. 드랍/취소 → 제외
2. 제출일이 weekStart 이후 또는 없음 → 진행중
3. 발표/면접일:
   - 공란/추후 → 진행중
   - 서면 → 개찰 진행
   - 날짜가 weekStart 이후 → 진행중
4. 개찰일:
   - 공란/추후 → 개찰
   - weekStart 이전 → 제외
   - weekStart 이후 → 개찰
```

**날짜 처리 주의 (`parseLocalDate`)**
`new Date("2026-06-30")`은 UTC로 파싱되어 한국(UTC+9)에서 하루 전날이 됨.
→ 반드시 `new Date(year, month-1, day)` 방식 사용.

**날짜 셀 메모 툴팁**: 제출일·발표/면접일·개찰일에 메모 있으면 주황색 `●` + hover 표시.

**HWPX 생성**: `/api/hwpx` → kordoc 라이브러리 + `lib/templates/` 파일 사용.

## 4. 달력 컴포넌트 (`WeeklyCalendar`)

재사용 가능한 월 달력. 메인 대시보드와 향후 필요 시 다른 페이지에서도 사용.

**Props**
```typescript
{
  week: string              // ISO 주차 "2026-W26"
  performing: PerformingProject[]
  notes?: Record<string, Record<string, string>>  // name → field → note
}
```

**이벤트 색상 코드**
- 파란색 `#1d4ed8`: 제출일
- 주황/갈색 `#b45309`: 발표/면접일
- 초록색 `#15803d`: 개찰일

**메모 툴팁**: 이벤트에 `notes[프로젝트명][필드]` 값 있으면 hover 또는 탭(터치) 시 fixed 위치 툴팁 표시.

**모바일 최적화**
- 헤더의 범례(제출/발표/개찰) 숨김 → 월 네비 잘림 방지
- 이벤트 칩 글자 축약: "제출 XXX" → "제/발/개" 단일 글자
- 하루 최대 2개 칩 표시 → 초과 시 `+N` 뱃지
- 툴팁: `onTouchStart`로 탭 시 토글 표시, 외부 터치 시 자동 닫힘

## 5. 미래봇 (AI 챗봇)

**API Route**: `POST /api/chat`
- Gemini API 호출 (Function Calling)
- 프로젝트 DB 조회/수정 기능 포함
- DB 수정 시: `type: 'confirm'` 응답 → 사용자 확인 → `POST /api/chat/execute`

**날짜 인식**
- 매 요청마다 서버에서 `Asia/Seoul` 기준 오늘 날짜를 시스템 프롬프트에 주입
- 프로젝트 데이터에 `submit_date`, `interview_date`, `bid_date` 포함 → "오늘 개찰", "이번 주 발표" 등 날짜 기반 질문 가능

**기능 범위**
- 프로젝트 현황 조회 (날짜 기반 포함)
- 수주 분석
- 새 프로젝트 등록
- 기존 프로젝트 수정

## 7. 모바일 반응형

**기준**: 768px 이하 → 모바일 레이아웃

**SidebarContainer**
- 데스크톱: 사이드바 220px 고정 노출
- 모바일: 상단 48px 고정 바 + `☰` 햄버거 버튼 → 슬라이드 오버레이 사이드바

**메인 대시보드 (모바일)**
- 금주일정 → 달력 → 미래봇 순 세로 스크롤

**주간보고 (모바일)**
- 헤더 버튼 텍스트 축약 ("주간 HWPX" → "주간")
- 요약 카드 4열 → 2열
- 교육/기타 섹션 2열 → 1열

**달력 (모바일)**
- 헤더 범례 숨김, 월 네비만 표시
- 칩 텍스트 축약 (제/발/개), 하루 최대 2개 + `+N`
- 툴팁 탭 토글

**메모 팝업 (프로젝트 List)**
- 팝업 위치: `position: fixed` 사용, `window.scrollY` 제거
- 화면 하단 공간 부족 시 버튼 위로 자동 flip
