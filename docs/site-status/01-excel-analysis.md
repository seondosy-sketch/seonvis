## 대상 4개 시트의 열 구조 (공통)

헤더 행("현 장 명" 포함 행)을 탐지해 그 아래부터 파싱. B열 번호가 숫자인 행만 현장 데이터.

| 열 | 내용 | 비고 |
|---|---|---|
| A | (빈 번호 열) | |
| B | 번호 | 시트별로 1부터 다시 시작 |
| C | 현장명 | 줄바꿈 포함 다수 (예: "LH 인천송림4\n주거환경개선사업(전기)") |
| D | 감리원(사무원) | 여러 명 줄바꿈 나열 + 괄호(사무원) 혼재 → **원본 텍스트 그대로 보존** |
| E | 현장유선번호(단장 H.P) | 혼합 형식: "(010-…)" 괄호=단장 HP, 그 외 유선/개인별 번호 줄바꿈 |
| F | 시공사 | "추후기재" 등 포함 |
| G | 현장주소 + 사무실주소 | **한 셀에 "현) …" / "사) …" 마커로 병기** — 분리 가능 |
| H | 착수일 | datetime 객체와 "2026.06.18" 문자열 혼재 |
| I | 준공예정일 | 대부분 "2027.02.01" 문자열 |
| J | 비고 | |

건진법 시트의 dims가 HY열(233열)까지지만 실제 데이터는 J열까지 — 잔여 서식.

## 시트별 현장 수 — 정밀 재검증 (2026-07-15, 사용자 지적으로 재분류)

최초 분석은 "번호(B열)가 숫자인 행"만으로 집계해 **91건**이라고 보고했으나, 그중 2건은
번호만 있고 다른 필드가 전부 빈 **껍데기 행**이었다. 실질 데이터(감리원/전화/시공사/주소/
날짜 중 하나라도) 보유 여부까지 검증한 정확한 분류는 다음과 같다.

| 분류 | 건수 | 내용 |
|---|---|---|
| **유효 현장** (번호 있음 + 실질 데이터 있음) | **89** | 이관 대상 |
| 빈 껍데기 행 (번호는 있으나 다른 필드 전부 공란) | 2 | 아래 참고 |
| 번호 없는 이상 행 | 2 | 아래 참고 |
| **이름 있는 전체 행** | **93** | 89+2+2 |

**유효 현장 89건의 시트별 분포**: 건진법 67 / 주택법 6 / 건축법 2 / 전통소 14 —
**엑셀 상단 요약 셀("총 89개, 건진법67·주택법6·건축법2·전통소14")과 정확히 일치**.
최초 보고했던 "요약 셀 미갱신" 추정은 틀렸다 — 요약 셀이 맞았고, 최초 집계 스크립트가
빈 껍데기 행 2건을 실제 현장으로 잘못 셌던 것이 원인이었다.
