# 현장 현황 — 요구사항 정리

> 사이드바 「사무 보조 → 현장 현황」(현재 준비중) 활성화. 월별 기술인 배치이력이 아니라
> **현재 관리 중인 현장 기본정보를 빠르게 검색·확인하는 현장 목록** 기능.

## 범위

**관리 항목**: 현장명, 법 구분, 감리원/담당자, 현장 연락처, 책임자 핸드폰, 시공사,
현장주소, 사무실주소, 지역(자동 추출), 착수일, 준공예정일, 진행 상태(자동+수동), 비고,
즐겨찾기, 활성 여부.

**명시적 제외** (사용자 지시): 월별 현장현황·기준 월 선택·월별 배치 기술인·투입/철수
비교·기술인 이동 이력·월별 스냅샷·배치 조직도·월별 시트 동기화. 숨김 월별 시트 14개는
분석·이관·설계 전 과정에서 미사용.

## 메뉴/라우트

- 라우트 **`/sites`** — 기존 평면 라우트 규칙(`/projects`, `/engineers` 등)에 맞춤.
- 사이드바 「사무 보조」의 「현장 현황」에 `key: 'sites'`, `href: '/sites'` 부여 → 준비중
  해제. `lib/menuConfig.ts`에 `sites` 키 추가로 **항목별 권한(쓰기/읽기/숨김) 자동 연동**.

## 기존 프로젝트 List와의 차이

프로젝트 List(`projects`)는 **입찰 현황**(공고~개찰 lifecycle, 점수/낙찰 관리)이고,
현장 현황(`sites`)은 **수주 후 운영 중인 현장의 연락처·기간 대장**이다. 데이터 성격·수명이
달라 테이블·화면 모두 독립으로 간다 (향후 "수주 → 현장 전환" 연계는 확장 아이디어로만).

## 진행 상태 (요구 7)

착수일·준공예정일 기준 자동 계산 + **수동 지정(`manual_status`) 우선**:

```
manual_status 있으면 → 그 값
착수일·준공예정일 둘 다 없으면 → 일정 미등록
오늘 < 착수일 → 착수 전
오늘 > 준공예정일 → 준공 완료
준공예정일까지 90일 이하 → 준공 임박
그 외 → 진행 중
```

수동 지정 옵션: 자동(기본) / 착수 전 / 진행 중 / 준공 임박 / 준공 완료 / 중지 —
실제 준공·중단이 예정일과 다른 경우를 위해.

## 1차 MVP

포함: 메뉴 활성화, `/sites` 페이지, 목록 테이블, 4개 시트 91건(이상 행 2건 제외 시 89건 —
07 문서 ①②) 최초 이관, 통합 검색(전화 하이픈 무시), 법구분/진행상태/지역/시공사/준공시기
필터+초기화, 정렬, 상세 모달(복사 4종+토스트), 등록/수정/비활성화/삭제, 진행상태
자동+수동, 데이터 이상치 보고, 항목별 권한 연동.

제외(확장): 엑셀 동기화 UI(구조만 예약 — 05 문서), Excel 내보내기, 지도 연동,
프로젝트 List 연계, 카드 보기.
