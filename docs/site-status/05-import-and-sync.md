# 현장 현황 — 최초 이관·향후 동기화 (2026-07-15 확정 반영)

## 최초 이관 절차 (구현 시 1회 실행, 개인정보는 저장소에 남기지 않음)

engineers와 같은 방식이되 **PII 보관 원칙을 강화**: 파이썬 스크립트가 `Project
Portfolio.xlsx`를 세션 스크래치패드에서 직접 읽어 Supabase REST로 배치 INSERT하고,
스크립트·산출 SQL은 리포지토리에 남기지 않는다(`migration_sites.sql`은 스키마만).

### 파싱 규칙 (01 문서 정밀 재검증 결과 반영)

1. **대상 시트 4개만 연다** (건진법·주택법·건축법·전통소). 숨김 시트는 열지도 않는다.
2. "현 장 명" 텍스트가 있는 헤더 행 탐지, 그 아래부터 읽는다.
3. **번호(B열)가 숫자이고, 감리원/전화/시공사/주소/착수일/준공예정일 중 하나라도
   값이 있는 행만** 유효 현장으로 인정 (01 문서: 이 기준으로 89건, 엑셀 자체 요약과 일치).
   - 번호가 없는 행(서울사무소, 천안신부 잔여) → 제외 목록.
   - 번호는 있으나 다른 필드가 전부 공란인 행(주택법#8·건축법#8 "역세권시도시" 쌍) →
     제외 목록("확인 필요" — 실질 데이터가 없어 사이트로 볼 근거 부족).
4. **현장명**: 원본(`original_site_name`) 보존 + 줄바꿈→공백 정규화(`site_name`).
5. **법 구분**: 시트명 → `source_category`(원본 그대로) + `legal_category`(표준 표시값,
   04 문서 매핑표).
6. **주소 분리**: G열 `현)` 뒤 → site_address, `사)` 뒤 → office_address.
7. **지역**: site_address에 `extractRegion` 적용.
8. **연락처**: E열 원본을 `site_phone_raw`에 그대로 보존. 정규식으로 유선(지역번호 패턴)과
   010 패턴을 분리 추출해 `site_landline`(여러 개면 `; ` 이어붙임) / `manager_mobile`(첫
   010)에 넣는다. 번호가 2개 이상이거나 패턴이 애매하면 `phone_uncertain = true`.
9. **날짜**: datetime 객체는 그대로, 문자열은 `YYYY[./-]M[./-]D` 패턴 파싱. 실패 시 null +
   오류 목록(01 문서 재검증에서 실패 0건 확인).
10. 전 현장 `active=true`, `manual_status=null`(자동), `is_favorite=false`.

### 이관 보고 (요구 11·14 — 건수만, 전체 개인정보 미노출)

이관 건수 / 제외 건수(사유별) / 오류 건수 / 중복(확인 필요) 건수만 채팅·문서에 보고한다.
개별 현장의 전화·주소 원문은 보고에 포함하지 않는다(구조적 이상 행 2~3건처럼 사용자
판단이 꼭 필요한 경우만 예외적으로 이름 수준까지 언급 — 이미 01 문서에 기록됨).

## 향후 엑셀 동기화 (MVP 미구현 — 구조 예약)

「파일 선택 → 표시 시트 4개만 읽기 → 기존 DB 비교 → 미리보기 → 사용자 확인 → 반영」.
숨김 월별 시트는 동기화에서도 제외.

### 동일 현장 판정 우선순위 (요구 10, 09 문서 ⑨로 확정)

1. **엑셀에 `site_code` 값이 있으면** 그 값으로 매칭(향후 내보내기 파일을 수정해 재이입한
   경우 — 정확 매칭). 현재 엑셀엔 없는 값이라 최초 동기화 라운드에서는 이 단계가 걸리지
   않고 2단계로 내려간다.
2. `site_code`가 없으면 **정규화 현장명 + `source_category` + 주소** 조합으로 비교
   (legal_category가 아니라 source_category로 비교 — 원본 시트 체계가 동기화 매칭
   기준이므로).
3. 2에서 애매하면 **시공사·공사기간(착수~준공예정일)을 보조 비교값**으로 사용.
4. 그래도 불확실하면 자동 판정하지 않고 "동일 현장 판단 불가"로 사용자 확인.

### 비교 결과 상태 7종 (요구 12)

변경 없음 / 신규 현장 / 기존 정보 변경(항목별 전→후 표시) / **새 엑셀에 없는 기존 현장**
(자동 종료·비활성·삭제 금지 — "그대로 유지 / 종료 후보 / 비활성 처리" 중 사용자 선택) /
중복 후보 / 동일 현장 판단 불가 / 데이터 오류.

### 반영 시

`site_sync_logs`에 실행 이력 기록. 반영은 삭제 없이 INSERT/UPDATE/`active` 변경만.
향후 Excel 내보내기 구현 시 `site_code`를 반드시 포함해, 그 파일을 다시 가져올 때
1단계(정확 매칭)로 동기화되게 한다.
